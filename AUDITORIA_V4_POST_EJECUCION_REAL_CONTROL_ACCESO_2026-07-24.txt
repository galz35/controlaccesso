C-02. REPORTE SIN FILTRO SIGUE EXPONIENDO TODOS LOS EDIFICIOS

Severidad: CRÍTICA.

Código:

  let edificioId = query.edificioId;
  if (query.edificioId) {
    edificioId = resolveBuilding(req.user, query.edificioId);
  } else {
    resolveBuilding(req.user, undefined);
  }

En el else se descarta el valor retornado.

Para un registrador, resolveBuilding retorna el edificio asignado, pero
edificioId permanece undefined. El servicio consulta todos.

Corrección:

  const edificioId = resolveBuilding(
    req.user,
    query.edificioId,
  );

  return this.acceso.reporte(
    edificioId,
    query.tipoPersona,
    query.desde,
    query.hasta,
    query.pagina,
    query.porPagina,
    query.motivoAcceso,
  );

Pruebas obligatorias:

- CPF 121 con edificio 1 recibe 403.
- admin sin filtro obtiene todos.

con datos reales.

- guardar solo <uuid>.webp en FotoUrl o crear PhotoId;
- Dashboard usar basename validado;
- assertPhotoAccess buscar el registro y resolver edificio;
- verificar existencia;
- admin puede ver todos;
- CPF solo su edificio.

Ejemplo:

  async assertPhotoAccess(fileName: string, user: any) {
    const pool = await this.db.getPool();
    const result = await pool.request()
      .input('FileName', fileName)
      .execute('sp_Acceso_FotoAutorizar');

    const row = result.recordset[0];
    if (!row) throw new NotFoundException();

    resolveBuilding(user, row.EdificioId);
    return this.photoDirectory;
  }

C-05. MODELO TIPO MOVIMIENTO NO ES REPRODUCIBLE NI COHERENTE

Severidad: CRÍTICA DE DATOS.

Base activa:
- tblRegistroAcceso.TipoMovimiento existe, VARCHAR(20), NOT NULL.
- 31 movimientos normales se observan como ENTRADA.
- 3 salidas independientes conservan SALIDA_NOCTROL.
- 1 salida independiente todavía tiene FechaSalida NULL.

Git:
- ningún script crea ni agrega TipoMovimiento;
- 007 lo usa directamente en sp_Acceso_RegistrarSalida;
- una instalación limpia fallará o quedará distinta.

007 salida normal:

  TipoMovimiento = 'SALIDA_EDITOR'

El valor no está definido en el negocio y parece un typo.

007 salida independiente:
- inserta TipoPersona SALIDA_INDEPENDIENTE;
- no asigna TipoMovimiento;
- dependerá del default no versionado, probablemente ENTRADA.

Resultado:
Una salida independiente nueva puede quedar tipada como ENTRADA.

Corrección:
Definir primero el modelo.

Opción compatible:

  ACCESO_NORMAL | SALIDA_SIN_ENTRADA.
- FechaEntrada/FechaSalida conservan evento.

Opción correcta:
Tabla separada de movimientos:
- ENTRADA;
- SALIDA;
- NOVEDAD.

No migrar los 3 registros históricos automáticamente sin revisión.

Migration debe:
- crear columna si falta;
- crear default con nombre estable;
- crear CHECK;
- usar valores documentados;
- actualizar SP;
- incluir prueba de instalación limpia.

A-02. RATE LIMIT SIGUE SIENDO POR IP, NO POR USUARIO

Middleware corre antes de AuthGuard. En ese momento req.user normalmente no
existe. identity termina siendo req.ip.

Aunque el código intenta username/carnet, no lo obtiene.

Con trust proxy=true y puerto 3001 escuchando en todas las interfaces:
- un cliente directo puede manipular X-Forwarded-For;
- puede evadir límites;
- o crear muchas entradas en Map.

Configurar:

  app.set('trust proxy', 1);

solo si hay exactamente un proxy confiable.

Preferir @nestjs/throttler:
- login por IP: 10/min;
- API por sub/carnet: 300/min;
- Redis si hay más de una instancia.

12. CÓDIGO RÁPIDO RECOMENDADO
======================================================================

12.1 Reporte

Reemplazar todo el if:

  const edificioId = resolveBuilding(
    req.user,
    query.edificioId,
  );

12.2 Salida normal

Controller:

  const edificioId = resolveBuilding(req.user, undefined);
  return this.acceso.registrarSalida(
    id,
    edificioId,
  );

Servicio:

  async registrarSalida(
    id: number,
    edificioId?: number,
  ) {
    return pool.request()
      .input('Id', id)
      .input('EdificioIdAutorizado', edificioId ?? null)
      .execute('sp_Acceso_RegistrarSalida');
  }

Para diferenciar admin, controller puede pasar undefined únicamente cuando
rol=admin. Registrador siempre debe pasar entero.

12.3 Foto

Guardar:

  return fileName;

No:

  return `fotos_acceso/${fileName}`;

Para históricos:

  const fileName = fotoUrl.split('/').pop();

validar regex antes de usar.

12.4 Curso DTO

  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  duracionHoras?: number;

Evento:

  @Type(() => Number)
  @IsInt()
  cursoId: number;

12.5 Enter

  onKeyDown={(event) => {
    if (event.key !== 'Enter') return;
    event.preventDefault();
    void buscar();
  }}

12.6 Login

Renderizar submit solo para:
- externo;
- empleado con allowDev.

No mostrar dos acciones rojas simultáneas para Portal.

12.7 Health

Backend:

  if (!databaseOk) {
    throw new ServiceUnavailableException({
      status: 'error',
      database: 'disconnected',
    });
  }

Script:

  body=$(curl --fail --silent "$API_URL") || exit 1
  jq -e '.status=="ok" and .database=="connected"' \
    <<< "$body" >/dev/null || exit 1

12.8 Backup

No hardcodear credencial.
No usar eval.

Variables:
- SQL_HOST;
- SQL_USER;
- SQL_PASSWORD desde archivo 600/secret;
- BACKUP_CONTAINER_PATH;
- BACKUP_HOST_PATH montado.

Usar una orden explícita y comprobar:
- archivo existe;
- tamaño >0;
- RESTORE VERIFYONLY;
- checksum;
- copia externa.
